# Specification: エントリーポイントの分散と属性入力の各ページへの移設

## 概要
現在の `index.html` 中心のエントリー方式を変更し、ユーザーの参加パターンに合わせて、各コンテンツページ（事前アンケート、ゲーム、事後アンケート）の冒頭で必要に応じて属性情報（匿名コード、性別、年齢）を入力・確認できる仕組みを構築する。`index.html` はシンプルなナビゲーションページとする。

## 機能要件

### 1. `index.html` (ランディングページ)
- **入力削除:** 既存の「匿名コード」「性別」「年齢」入力フォームおよび関連ボタンを削除する。
- **ナビゲーション:** 「事前アンケート」「ゲーム」「事後アンケート」へのリンクボタンのみを表示し、常時クリック可能とする。

### 2. 共通入力コンポーネントの導入
- **対象ページ:** `pre-survey.html`, `game.html`, `post-survey.html`
- **挙動:**
    - ページ読み込み時、ブラウザ（LocalStorage等）に保存済みの属性情報（匿名コード、性別、年齢）があるかチェックする。
    - **情報がない場合:** ページの冒頭（または開始前）に属性情報の入力フォームを表示し、入力を必須とする。入力完了後、情報を保存し、本来のコンテンツを開始する。
    - **情報がある場合:** 保存済みの情報を表示（例：リードオンリーのフィールドや確認メッセージ）し、そのままコンテンツを開始できる状態にする。

### 3. 各ページの具体的変更
- **`pre-survey.html`:** 冒頭に入力フォームを追加。入力データは保存され、後続のゲーム等で再利用される。
- **`game.html`:** ゲーム開始画面に入力フォームを追加。情報未入力時のみ表示・入力を促す。
- **`post-survey.html`:** 冒頭に入力フォームを追加。情報未入力時は入力を促し、入力済みの場合は既存情報を表示する。

### 4. データ送信
- **タイミング:** 既存の送信タイミング（アンケート回答完了時、ゲーム終了時など）を維持する。
- **データ内容:** 送信時に `participant.js` (LocalStorage) から最新の属性情報を取得し、送信データに含める。

## 非機能要件
- **データ継続性:** 一度入力された属性情報はブラウザに保持され、他のページ移動時に引き継がれること。
- **UI一貫性:** 各ページに追加する入力フォームは、既存のデザイン（フォント、配色）と統一感を持たせること。

## 受け入れ条件
- [ ] `index.html` がボタンのみのシンプルな画面になっていること。
- [ ] 各コンテンツページ（事前・ゲーム・事後）を新規で開いた際、属性入力が求められること。
- [ ] 一度属性を入力すれば、他のページでは再入力が不要（または入力済み表示）であること。
- [ ] 既存のデータ送信機能が、入力された属性情報を正しく含んで実行されること。
